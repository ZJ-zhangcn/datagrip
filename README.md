# 保险核心业务 SQL 脚本库

## 1. 项目简介

本仓库是保险核心业务系统的 SQL 脚本集合，用于数据处理、业务支持、问题排查和需求实现。所有脚本均存放于 `sql/` 目录下，旨在为开发、测试和运维人员提供一个标准、可复用的脚本资源库。

## 2. 目录结构

```
.
├── .idea/         # IDE 配置文件
├── sql/           # 存放所有 SQL 脚本
└── README.md      # 本说明文档
```

所有业务相关的 SQL 脚本都集中管理在 `sql/` 目录中。

## 3. 脚本分类

`sql/` 目录下的脚本主要根据业务功能或需求编号进行命名，可以大致分为以下几类：

### 3.1. 按业务模块分类

这类脚本通常以中文命名，直接描述其业务功能。

- **保单服务**:
  - `保单登记-核心-清空temp表.sql`
  - `保单失效状态.sql`
  - `购买交清保额.sql`
- **收付费与财务**:
  - `财务提数-上海人寿.sql`
  - `收付费模拟.sql`
  - `续期回退.sql`
  - `银保手续费账务规则调整.sql`
- **理赔与满期**:
  - `理赔.sql`
  - `满期回退.sql`
  - `满期终止保单理赔操作优化需求.sql`
- **产品与分红**:
  - `红利.sql`
  - `万能月结费用计算.sql`
- **渠道支持**:
  - `银保通.sql`
  - `经代通.sql`
  - `官微.sql`

### 3.2. 按需求/问题编号 (REQ/BUG) 分类

这类脚本与特定的开发需求或缺陷修复任务关联，通常以 `REQ-` 或 `BUG-` 作为前缀。

- **示例**:
  - `REQ-3546+REQ-3936保盾云.sql`: 对应需求 REQ-3546 和 REQ-3936 的相关脚本。
  - `BUG-958-建行续期扣费回盘异常.sql`: 对应缺陷 BUG-958 的修复脚本。

## 4. 使用说明

1.  **查找脚本**: 根据您需要处理的业务模块或对应的需求/缺陷编号，在 `sql/` 目录下查找相关文件。
2.  **理解上下文**: 在执行任何脚本之前，请务必打开文件，仔细阅读其中的注释和逻辑，确保您完全理解其功能和潜在影响。
3.  **在测试环境验证**: **严禁**在未经验证的情况下直接在生产环境中执行任何脚本。所有脚本都必须首先在测试环境中执行并验证结果。
4.  **参数替换**: 许多脚本可能包含需要替换的占位符参数（如保单号、日期等），请根据实际情况进行修改。
5.  **备份数据**: 对于任何数据修改（`UPDATE`, `DELETE`, `INSERT`）操作，强烈建议在执行前备份相关数据。
